# Humi 可收起运行侧栏设计

## 目标

让 Humi 的默认界面以对话为中心，结构接近 Codex：

- 消息区占满剩余空间并独立滚动。
- 输入框稳定贴近窗口底部，不再留下无意义的大块空白。
- 右侧“运行状态”侧栏默认收起，通过右上角按钮显隐。
- 用户的展开选择只影响当前 Humi 页面，不写入本地存储。

## 已选方案

采用方案 A：条件渲染现有 `HumiOperationsRail`，并重排 Humi 对话区。

不改 Hub 左侧的 Humi、Hype、Hush、Hexa 导航，也不增加新的全屏模式。侧栏按钮只控制 Humi 已有的右侧运行状态区域。

## 交互

1. 每次挂载 Humi 时，运行侧栏为收起状态。
2. 对话区右上角显示一个 Codex 风格的侧栏按钮，使用清晰的面板图标。
3. 收起时按钮标签为“展开运行状态”，`aria-expanded=false`。
4. 点击后显示现有运行状态侧栏，按钮标签改为“收起运行状态”，`aria-expanded=true`。
5. 再次点击立即收起侧栏。
6. 切换到其他房间再返回 Humi，或重新启动应用后，侧栏重新保持默认收起。

隐藏状态不保留可聚焦的侧栏控件，避免键盘用户进入不可见区域。侧栏收起不停止现有只读状态获取；展开后可以立即看到当前数据。

## 布局

### 对话区

`humi-conversation-stage` 改为上下两行的网格：

- 第一行是 `minmax(0, 1fr)` 的消息区，负责滚动。
- 第二行是正常文档流中的输入框。

输入框不再使用绝对定位。容器保留紧凑的底部安全间距，使输入框视觉上贴底但不碰到窗口边缘。详情面板继续锚定在输入框上方。

### 运行侧栏

- 收起时 `humi-workspace` 只有一个内容列，对话区占满宽度。
- 桌面宽度下展开时恢复“主区 + 252px 侧栏”。
- 窄窗口下展开时侧栏作为右侧浮层出现，不压缩对话到不可用宽度。
- 显隐使用轻量的透明度和位移动效；系统要求减少动态效果时关闭动效。

### 控制按钮

按钮放在 Humi 对话区右上角，不占消息正文宽度。视觉采用现有柔和白色、细描边和小圆角语言，与用户给出的 Codex 控件参考一致。

## 组件边界

- `HumiModule` 持有一个非持久化的 `operationsOpen` 布尔状态，初始值为 `false`。
- 新的侧栏按钮只负责切换此状态。
- `HumiOperationsRail` 保持现有数据和操作接口，不修改业务行为。
- CSS 根据打开状态类名控制桌面双栏和窄屏浮层。

## 错误与边界

- 运行状态加载失败仍沿用现有提示，不能影响对话输入。
- 隐藏侧栏时仍允许 Humi 正常聊天、显示加载态和打开详情。
- 长消息必须在消息区滚动，不能把输入框推出窗口。
- 输入框增长到最大高度后内部滚动，不能重新制造底部空白。

## 测试与验收

- 渲染 Humi 后，侧栏默认不存在，按钮为未展开状态。
- 点击按钮后侧栏出现，按钮标签和 `aria-expanded` 正确变化。
- 再次点击后侧栏消失。
- 不读写 `localStorage` 或 `sessionStorage` 保存显隐状态。
- 样式测试确认对话区使用“滚动消息区 + 正常流输入框”，不再依赖输入框绝对定位。
- 在桌面宽度和窄窗口下进行视觉检查，确认对话内容不被侧栏遮挡、输入框下方不再有明显空白。
