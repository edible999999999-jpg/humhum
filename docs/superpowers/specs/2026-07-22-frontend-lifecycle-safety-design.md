# 前端生命周期与权限安全修复设计

## 问题

当前主分支存在几类会直接破坏用户信任的前端问题：关闭阻塞提问会被当成允许、权限提交失败后弹窗仍会消失、Humi 在切换 Hub 模块时会丢失正在生成的回复、旧桌宠定时器会清除新事件、Hush 状态命令可能并发执行，以及非 Tauri 预览会在启动阶段白屏。

## 已选方案

采用聚焦修复，不重构全局状态架构：

- 阻塞提问的关闭动作明确提交拒绝；提交失败时保留弹窗并允许重试。
- 权限回答失败时保留确认弹窗，不用定时器隐藏错误。
- Hub 切换模块时保留 Humi 实例，只切换可见性，确保运行中的回答和聊天状态继续存在。
- 桌宠状态定时器采用“新事件取消旧定时器”，卸载时统一清理。
- Hush 状态刷新使用单次请求互斥，事件触发刷新并保留 60 秒兜底轮询。
- 非 Tauri 环境跳过桌面窗口 API，让浏览器预览能正常进入 Hub。
- 为本轮触达的无标签输入与图标按钮补充可访问名称，恢复明确的键盘焦点。

## 数据与错误语义

- 用户关闭阻塞提问等价于 `deny`，绝不等价于 `allow`。
- IPC 失败只改变当前弹窗的错误状态，不改变后端请求生命周期，也不自动 dismiss。
- Humi 的生成请求不因 UI 分区切换而取消；离开整个 Hub 后仍按现有卸载语义处理。
- Hush 同一时间最多有一个状态查询；并发触发复用当前查询或直接跳过本轮。

## 验收

- 权限与提问弹窗的成功、失败、关闭语义有回归测试。
- Humi 在模块切换后保持挂载，返回时可看到生成完成的回复。
- 新桌宠事件不会被旧事件定时器清除。
- Hush 轮询不会重叠，界面显示 1 分钟刷新语义。
- 普通浏览器环境不因 Tauri API 抛错而白屏。
- 完整前端测试与 `npm run build` 通过。
