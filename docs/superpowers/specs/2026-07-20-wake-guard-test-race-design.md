# Wake Guard 重启测试竞态修复设计

## 问题

`enabled_guard_restarts_after_its_child_exits` 启动 `/bin/sleep 0.05`，固定等待 100ms 后就要求 PID 已变化。在负载或进程回收调度稍慢时，子进程虽然已经退出，但 Tokio 尚未稳定观察到退出状态，测试会误报生产自愈失败。

诊断确认 100ms 附近的子进程已进入 `Z <defunct>` 状态；加入一次系统状态采样后原测试通过，证明失败来自固定等待竞态。

## 已选方案

- 不修改 `WakeGuardState` 的生产逻辑。
- 删除临时 `ps` 诊断。
- 测试最多等待 2 秒，每 10ms 调用一次 `reconcile_desired_state(true)`。
- 当返回的 PID 与首次 PID 不同时立即成功。
- 超过 2 秒仍未重启时，让测试带明确超时信息失败。
- 最后仍显式关闭 guard，避免遗留测试子进程。

## 验收

- 修复前的原测试能稳定复现失败。
- 条件轮询版测试连续运行至少 10 次全部通过。
- 完整 Rust 测试通过，除既有忽略项外没有失败。
- Hush 修复继续以原隔离分支和计划执行。
