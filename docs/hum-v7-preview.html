1. Session Mapping 会话连接 · 吸收 Agent

设计思路 · 吸收

Hum 直接"吃掉"连接的 Coding Agent：

吸收 (absorbed) — Agent 图标作为发光能量球漂浮在伞盖体内。颜色 = Agent 品牌色。像水母吞噬了猎物一样，在半透明的身体里隐约可见。

触手 (tentacles) — 下方触手数量随会话增多而变多，作为会话量的视觉 hint。用户点击时更明显。

映射规则

0 个会话 → 空灵的水母，体内无光
1 个会话 → 体内出现 1 个 Agent 光球
2 个会话 → 2 个光球轻轻漂浮
3 个会话 → 3 个光球
≥4 个 → 触发幼体模式（压力返老还童）

新 Agent 接入时，光球从底部"被吸入"；断开时光球"被消化排出"。光球在伞盖内缓慢漂浮，像消化中的食物。

Agent Creature Designs 吸收态生物 · 每个 Agent 的形态

吸收态设计

每个 Coding Agent 在 Hum 体内是一只小型深海生物——被水母吃掉后安静漂浮在伞盖中。

平时 — 安静隐匿，低透明度，不遮挡水母本体
有事件时 — 发光亮起，让用户注意到是哪个 Agent 在活动
风格 — 平滑矢量、渐变填充、多层透明度、可爱但不幼稚

3. Juvenile Mode 灯塔水母 · 返老还童

灯塔水母的秘密

灯塔水母是地球上唯一能"返老还童"的生物。遭遇压力时，它会逆转回幼体重新生长——永生循环。

Hum 继承了这个能力：当管理的会话 ≥ 4 个时，工作压力触发返老还童，Hum 缩小为幼体形态。同样的外形，只是变小了——越忙越小越努力。

形态规则

成体 Medusa — ≤3 sessions
标准尺寸，优雅飘逸。

幼体 Polyp — ≥4 sessions
同样的外形，等比缩小约 65%。触角依然在，只是更密集。像一个缩小版的自己在拼命干活。

会话减少时会"长大"回成体，有平滑的缩放过渡动画。

4. Jet Propulsion Drag 喷水推进 · 拖拽交互

喷水推进

水母通过收缩伞盖喷水来移动——这是 Hum 的拖拽动效：

1. 拖拽中 — 伞盖快速收缩脉冲，从底部喷出水泡尾迹，方向与移动方向相反
2. 拖得越快 — 喷射越密集，气泡越多
3. 释放 — 最后一次喷水，然后慢慢飘停恢复呼吸
4. 幼体拖拽 — 同样的喷水，只是气泡更小
